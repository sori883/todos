# コードレビュー

コミットされていない変更の包括的なセキュリティ・品質レビュー:

1. 変更ファイルを取得: git diff --name-only HEAD

2. 各変更ファイルについて以下をチェック:

**セキュリティの問題（CRITICAL）:**
- ハードコードされた認証情報、APIキー、トークン
- 正当な理由のない `unsafe` ブロック
- 文字列フォーマットによるインジェクション脆弱性
- パストラバーサルリスク
- 安全でないファイル操作（権限、シンボリックリンク）

**コード品質（HIGH）:**
- 50行を超える関数
- 800行を超えるファイル
- 4レベルを超えるネスト深度
- src/ 配下の `.unwrap()` （`expect()` や `?` を使用すべき）
- `Result`/`Option` の適切なエラーハンドリング欠如
- `todo!()` / `unimplemented!()` が本番コードに残存
- パフォーマンスに影響する不要な `.clone()`
- 借用で済む箇所での所有権取得

**ベストプラクティス（MEDIUM）:**
- パブリックAPIの `///` ドキュメントコメント欠如
- 未使用の `use` 文（dead import）
- 不必要な `mut` 宣言
- 新しいコードのテスト欠如
- `#[allow(...)]` の濫用

3. レポートを生成:
   - 重要度: CRITICAL、HIGH、MEDIUM、LOW
   - ファイルの場所と行番号
   - 問題の説明
   - 修正の提案

4. CRITICALまたはHIGHの問題がある場合はコミットをブロック

セキュリティ脆弱性のあるコードを承認してはならない！
